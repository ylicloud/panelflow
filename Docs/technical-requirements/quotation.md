## 2. 列表逻辑

入口方法：

- `QuotationController.Index()`

核心规则：

- 支持关键字搜索：报价单编号、报价单名称、报价人、客户编号、客户名称、客户别名
- 默认仅显示近 2 年数据（可勾选“显示所有时间段”）
- 列表按 `fasj` 倒序，再按 `fabh` 倒序
- `dqzt` 显示规则（列表「当前状态」列）：
  - `0` -> 草稿
  - `1` -> 文字「(无内容)」：空报价单，尚无任何有效报价内容
  - `10` -> 已成立（绿色高亮）
  - 其他数值：原样显示数字
- “修改”“删除”“报价”按钮显示条件（三者相同，代码统一）：
  - `QuotationEditRules.CanOwnerOperate(bjr, dqzt, 当前登录用户名)` 为真
  - 等价于：`BJFAT.bjr` 与当前用户一致（忽略大小写、trim）且 `BJFAT.dqzt !== 10`（`EstablishedStatus`）
  - 实现：`PanelFlow.Core/Rules/QuotationEditRules.cs`；服务端校验见 `QuotationService.ValidateEditAccess`
- “复制”按钮：列表每行均可点（含已成立）；弹出对话框填写新编号，浅拷贝 `BJFAT`+`BJB`，见 §3.1
- “报价”按钮跳转 `QuotationController.Price()` → `GET /Quotation/Price?id={fabh}`，明细编辑能力按需迭代
- 结构维护页 `SearchQuotations` 检索下拉：同样仅返回 `CanOwnerOperate` 为真的报价单（`WhereOwnerOperable` 扩展）；写操作另含管理员例外，见 `quotation-element-maintenance` 需求 8.2

## 3. 新建逻辑

入口方法：

- `QuotationController.Create()` `GET/POST`

页面与默认值：

- 新建页输入项保留：报价单编号、报价单名称、客户搜索选择、备注
- 以下字段不在页面输入，由后端赋值：
  - `fasj = DateTime.Now`
  - `famxbh = 0`
  - `falx = 1`
  - `dqzt = 1`：新建保存后固定为 **1**，表示**空报价单**（尚无任何实质报价内容/尚未进入可提交状态）；与列表 `dqzt=1` 时显示「(无内容)」一致
  - `bjr = 当前登录用户名`
- 客户选择通过关键字检索 `KHYLB`（仅 `gsld` 非空），选择后回填只读框：
  - 客户编号 `gsbh`
  - 客户名称 `gsmc`
  - 别名 `gsld`

事务流程（必须全部成功）：

1. 校验 `BJFAT.fabh` 不能重复
2. 校验 `BJB` 中不能已有 `fabh=当前报价单编号` 的数据
3. 插入 `BJFAT` 新记录
4. 插入 `BJB` 默认行 1：
   - `fabh = 当前报价单编号`
   - `x_bm = 0`
   - 其他字段默认值
5. 插入 `BJB` 默认行 2：
   - `fabh = 当前报价单编号`
   - `x_bm = 9999`
   - `x_mc = 总计`
   - 其他字段默认值
6. 提交事务，返回创建成功

## 3.1 复制报价单（浅拷贝）

入口：

- 列表行「复制」按钮 → 弹窗 → `POST /Quotation/Clone`
- 服务：`QuotationService.CloneAsync`

规则：

- 任意列表可见的报价单均可复制（含 `dqzt=10` 已成立）；原单不改动
- 必填新 `fabh`；名称默认「原名称_副本」；客户/备注默认同源，可改
- 新单 `bjr=当前用户`，`fasj=现在`
- 新单 `dqzt`：源单有控制柜则为 `0`（草稿），否则 `1`；**从不**复制 `dqzt=10`
- 仅复制 `BJFAT` + 全部 `BJB` 行（`INSERT…SELECT` 换 `fabh`）
- **不复制**：汇总表、`XM_*`、合同、采购计划
- 成功后跳转新单详情页；超时策略与改号类似（600 秒）

## 4. 修改逻辑

入口方法：

- `QuotationController.Edit()` `GET/POST`

核心规则：

- 编辑页去除输入项：创建时间、当前状态、报价人、方案模型编号、方案类型
- 客户仍通过关键字搜索选择（3 个只读回填框）
- 编辑页“客户名称/别名”回显来源：
  - 不使用列表拼接值
  - 根据报价单的 `khbh` 到 `KHYLB.gsbh` 查询 `gsmc` 与 `gsld`
- 更新时仅更新允许修改字段（如报价单名称、客户编号、备注），其余字段保持原值
- **Edit GET/POST 权限**（`QuotationService.ValidateEditAccessAsync` / `UpdateAsync`）：
  - 仅报价人本人（`bjr` 与当前登录用户名一致）
  - `dqzt !== 10`（已成立不可编辑）
  - 不满足则拒绝访问或保存

### 4.1 纠错改号（独立入口）

详细规则见 `Docs/修改方案编号fabh.txt`。

- 编辑页**报价单编号默认只读**；满足条件时显示链接「编号录入有误？申请修改」，打开 Modal 提交
- 入口：`QuotationController.RenameQuotationNo()` `POST`
- 服务：`QuotationService.CanRenameFabhAsync()` / `RenameFabhAsync()`
- 允许改号条件（全部满足）：
  - `XMYLB.bjd_fabh` 不存在原编号关联
  - `dqzt !== 10`
  - `bjr` 为当前用户
  - `fasj` 距今天 ≤ 30 天
  - 新编号在 `BJFAT` 中不存在
- 同事务 UPDATE：`BJB` → `BJB_HZB` → `BJB_XMYJB` → `BJB_XMYJHZ` → `BJB_XMHZ` → `XM_YJB` → `XM_CKJS` → `XM_YJHZ` → `XM_HZ` → `BJFAT`
- **审计**：`ActionType=RenameFabh`，`Module=Quotation`，写入 `SYS_AUDIT_LOG`（成功/失败均记录，`BeforeData`/`AfterData` 含旧新编号与各表影响行数）

## 5. 删除逻辑

入口方法：

- `QuotationController.Delete()` `POST`
- `QuotationService.DeleteAsync()`

删除流程：

1. 前端确认提示：删除后无法恢复
2. 后端再次校验权限与状态（与列表 `CanShowRowActions` / 第 2 节按钮一致）：
   - 仅报价人本人可删（`bjr` 与当前登录用户名一致）
   - `dqzt === 10`（已成立）不可删；其余状态在满足本人条件下允许删除
3. 开启事务并按固定顺序删除：
   - 先删 `BJFAT` 对应 `fabh` 行
   - 再删 `BJB` 对应 `fabh` 行
4. 提交事务后返回删除成功

## 6. 关键约束汇总

- 报价单编号必须唯一
- 新建时 `BJFAT` 与 `BJB` 初始行必须同事务落库
- 新建时 `BJFAT.dqzt` 必须为 `1`（空报价单，含义见第 2、3 节）；`0` 草稿、`10` 已成立等用于后续业务阶段
- 删除时必须同时满足“本人 + 非已成立”条件（`dqzt !== 10`，与第 2 节列表展示一致）
- 删除时 `BJFAT`、`BJB` 必须同事务删除

## 7. 报价页面（导入元件 / 填写报价）

入口方法：

- `QuotationController.Price(id, section)`

页面结构：

- 子页面分为：
  - `section=import`：「导入元件」
  - `section=fill`：「填写报价」
- 默认进入规则：
  - `BJFAT.dqzt == 1`（空报价单）时，默认进入「导入元件」
  - `BJFAT.dqzt == 0`（草稿）时，默认进入「填写报价」
- 左侧目录树：
  - 根节点：当前报价单名称 `BJFAT.famc`（为空时回退显示报价单编号）
  - 子节点：`BJB` 中 `fabh=当前报价单` 且 `x_bm` 长度为 12 的行，显示 `x_mc`

### 7.1 导入元件（Excel -> Handsontable）

- 页面按钮：
  - 「打开excel元件表」
  - 「数据检查」
  - 「目录预览」
  - 「另存表格」（仅当前页面 Handsontable）
  - 「导出方案 Excel」（从库内 BJB 导出整单）
  - 「保存方案」
- 解析规则：
  - 只读取首个工作表
  - 跳过第 1 行标题
  - 只读取前 8 列（A-H）
  - 最多读取 5000 行有效数据
  - 遇到连续 5 行全空（前 8 列均空）即视为文档结尾，后续数据忽略
- 加载目标：右侧 `Handsontable`
- LicenseKey：`53ea8-c2678-49b80-cb40f-4dad4`
- 右键菜单：启用中文菜单，允许在线修改行列与单元格

#### 7.1.1 数据检查规则

点击「数据检查」时，按下列规则校验当前 Handsontable 数据（8 列）：

1. 第6列「数量」必须为正数
2. 当第3列「名称」与第4列「规格」同时为空时，第2列「单元号」必填
3. 数据需连续：两段非空数据之间若仅有空行，**连续空行不得超过 5 行**；1～5 个连续空行自动跳过、不报错（容错）；超过 5 个连续空行后再出现非空数据则判错
4. 第2列「单元号」不允许重复

校验结果显示在页面上方信息栏。未通过时，对 **出错单元格** 或 **出错整行** 进行浅红底 + 红框高亮（前端渲染层标记，不落库）。

#### 7.1.2 目录预览规则

点击「目录预览」时：

- **空报价单**（`dqzt==1` 且无柜）：直接根据当前 Handsontable 生成目录子节点（覆盖模式）。
- **非空报价单**：弹出三选一对话框：
  1. **覆盖原报价单内容**：清空左侧树后仅显示本次 Excel 解析出的节点；
  2. **追加到原报价单之后**：保留进入页时的原柜节点，再追加本次 Excel 新节点（新节点带「新增」标记）；
  3. **取消**：还原原树，不进入可保存状态。

预览节点由前端根据 **当前 Handsontable 数据** 生成（根节点仍为报价单名称）。

**数据来源**

- 仅处理 **第2列「单元号」非空** 的行（视为控制柜相关行）。
- 读取该行 **第6列「数量」**，解析为正整数 `N`；解析失败或为空时按 `N=1`。

**拆分规则（`N > 1` 时生成多条子节点）**

- 拆分后的展示名称列表长度应为 `N`（逻辑上等价于 `N` 台柜的预览节点）。
- **若单元号（用作预览名的基准字符串）末尾为 1～2 位数字**：  
  - 第 1 个节点名称保持原字符串不变；  
  - 后续 `N-1` 个节点名称：在末尾数字上递增，并 **保持末尾数字位数**（例如 `A01 → A02 → A03`）。
- **若末尾不是数字**：  
  - `N` 个节点名称分别为：`单元号 + "1"`、`单元号 + "2"`、…、`单元号 + "N"`（即原单元号作前缀，后缀从 1 递增到 N）。

**重复名称**

- 预览列表 **不去重**：若拆分或不同行产生相同展示名，**全部列出**。
- 统计全局展示名出现次数；出现次数 > 1 的节点标记为 **错误**：红色显示，悬浮提示 `错误:名称重复`。

**拆分节点样式**

- 由拆分产生的节点（且非重复错误）使用 **紫色** 醒目显示，悬浮提示 `已拆分`。

**点击子节点与表格跳转（重要）**

- 左侧每个子节点按钮携带 **`data-unit-no`**，值为该行 **第2列原始「单元号」**（未随拆分展示名改变）。
- 点击时在 Handsontable 中 **按第2列文本等于该单元号** 查找 **首次匹配行**，选中 **第2列单元格** 并滚动到可视区域。
- **未持久化**「拆分展示名 ↔ 行/单元格」的映射；跳转依据始终是 **原始单元号**。拆分后的紫色名称可与第3列「名称」等不一致，不影响跳转逻辑。

#### 7.1.3 另存表格 / 导出方案 Excel

**另存表格**（原「另存excel」）

- 仅将 **当前 Handsontable** 数据提交后端生成 `.xlsx`（与覆盖/追加模式无关，不含库内整单）
- 接口：`POST /Quotation/SaveImportExcel`
- 文件名：`报价元件表_{fabh}_{yyyyMMddHHmmss}.xlsx`

**导出方案 Excel**

- 从数据库 `BJB` 导出当前报价单全部控制柜及元件，格式与导入 8 列兼容，便于再导入或复用
- 接口：`GET /Quotation/ExportPlanExcel?id={fabh}`
- 柜标题行第 2 列写柜名称（`x_mc`），元件行第 2 列为空
- 文件名：`方案元件表_{fabh}_{yyyyMMddHHmmss}.xlsx`
- 空单（无柜）时按钮禁用

#### 7.1.4 保存方案规则（写入 BJB）

左侧工具栏「保存方案」用于将导入页当前“目录预览 + 表格内容”落库到 `BJB`。

**前端启用与前置条件**

- 按钮默认禁用，仅当以下流程成功后可点击：
  1. 已加载 Excel 数据；
  2. 「数据检查」通过；
  3. 「目录预览」成功并已选择 **覆盖** 或 **追加**。
- 保存前确认（按预览所选模式，各一次）：
  - 覆盖：`将覆盖原报价单全部内容…`
  - 追加：`将向原报价单追加 N 个控制柜…`
- 若用户在表格中做了任何编辑（改单元格 / 增删行），会自动使“目录预览结果”和“保存方案可用状态”失效，需重新执行「数据检查」与「目录预览」。

**提交接口与参数**

- 接口：`POST /Quotation/SavePlan`
- 防伪：必须携带 `RequestVerificationToken`
- 请求体（JSON）：
  - `fabh`：报价单编号
  - `tableJson`：当前 8 列表格二维数组（字符串化后的单元格值）
  - `treeNodeNames`：本次预览 **新增** 节点名称列表（不含追加模式下的原柜名）
  - `saveMode`：`overwrite`（默认）或 `append`

**后端保存前校验**

1. `fabh` 必填，且报价单必须存在；
2. 登录身份必须有效；
3. 权限控制：仅“报价单报价人本人”或“管理员”允许保存；
4. `tableJson` 不能为空；
5. `treeNodeNames` 不能为空（要求先执行目录预览）；
6. 根据表格解析出的“应拆分控制柜总数”必须与 `treeNodeNames.Count` 完全一致，否则拒绝保存（提示需重新目录预览）。

**写入行为**

- `overwrite`：删除该单除 `x_bm in ('0','9999')` 外的 BJB 行，再从柜码 `0001` 起重建。
- `append`：不删除已有柜/件；柜码从当前最大 4 位数字码 `+1` 起续编；仅插入本批 Excel 对应行。
- 保存过程以 NDJSON 流式回报进度 `(已完成/总数)`（按本批单元计）。

**表格解析与业务含义**

- 按行扫描 `tableJson`（仅取前 8 列）：
  - 当第2列「单元号」非空时，视为一个新的“源控制柜块”起点；
  - 该行第6列「数量」解析为拆分数 `SplitCount`（<=0 或非法时按 1）；
  - 后续直到下一个“第2列非空行”之前的行，若第3~7列任一列有值，视为该控制柜块下的元件行。
- 元件行字段映射：
  - `名称 <- 第3列`
  - `规格 <- 第4列`
  - `单价 <- 第5列(Decimal，失败按 0)`
  - `数量 <- 第6列(Decimal，失败按 0)`
  - `生产厂家 <- 第7列`

**落库策略（事务）**

1. 开启事务；
2. **overwrite**：先删除当前报价单在 `BJB` 中的旧明细：`x_bm NOT IN ('0','9999')`；**append**：跳过删除。
3. 按“控制柜拆分结果”写入：
   - **overwrite**：控制柜顺序号从 `0001` 起；**append**：从已有最大 4 位数字柜码 `+1` 起；
   - 控制柜名称 `x_mc` 取 `treeNodeNames` 对应顺序值；
   - 每个控制柜固定插入第 2 级目录节点（字典默认或内置 5 类）；
   - 元件明细编码规则：`{控制柜4位}0001{元件序号4位}`，`x_lx=11`；
   - 元件价格字段同步写入：`x_dj/x_bj_dj/x_bjb_bj/x_bjb_dj = 表格单价`。
4. 状态联动更新 `BJFAT.dqzt`：
   - 若本批写入存在控制柜且原状态为 `1`，更新为 `0`；
   - **仅 overwrite**：若本批无控制柜且原状态为 `0`，更新为 `1`。
5. 提交事务；任一异常回滚并返回失败。

**保存成功返回**

- NDJSON 完成事件含 `unitCount` / `componentCount` / `message`，前端信息栏展示。

### 7.2 填写报价

- 当前阶段页面布局与「导入元件」一致，后续再补充业务差异。
